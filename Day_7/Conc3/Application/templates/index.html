Welcome to the Flask Error Handling Demo!

This application demonstrates comprehensive error handling and debugging techniques in Flask.

Features Demonstrated:

  • Flask Debugger: Interactive debugging during development
  • Custom Error Pages: User-friendly error templates for different HTTP status codes
  • Custom Exception Classes: ValidationError and AuthenticationError
  • Comprehensive Logging: Different log levels for development and production
  • API Error Handling: JSON responses for API endpoints
  • Health Check Endpoint: For production monitoring

Try These Routes:

  • /debug?user_id=123 - Triggers Flask debugger when debug=True
  • /error/404 - Shows custom 404 page
  • /error/403 - Shows custom 403 page with authentication error
  • /error/400 - Shows custom 400 page with validation error
  • /error/500 - Shows custom 500 page
  • /simulate-crash - Randomly crashes to test error handling
  • POST /api/users - API endpoint with validation (send JSON with name and email)

API Testing Example:

curl -X POST http://localhost:5000/api/users \
  -H "Content-Type: application/json" \
  -d '{"name": "John Doe", "email": "john@example.com"}'

Important: Make sure FLASK_DEBUG=true to see the interactive debugger!
